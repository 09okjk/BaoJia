# 智能报价功能测试报告

## 1. 文档目的

本文档用于汇总当前智能报价系统的三轮功能测试结果，并给出当前阶段的功能可用性判断、上线前风险提示与后续建议。

测试目标不是证明“所有业务情况都已完美覆盖”，而是回答以下问题：

1. 当前主链路是否可运行
2. 当前 feedback 飞轮是否可运行
3. 当前 PDF 输出是否可运行
4. 当前模板识别与多模板渲染是否可运行
5. 当前系统是否已经达到“可继续试运行 / 可继续验收推进”的状态

## 4. 第一轮测试结果

第一轮重点验证“所有 skill 的标准入口是否可运行”。

## 5. 第二轮测试结果

### 5.1 测试内容

第二轮重点验证“主流程、feedback 飞轮与 PDF 主功能是否真正可用”。

执行内容：

1. `real-dimitra-m-input.json` 真实业务回归
2. `real-dimitra-m-accept-input.json` 显式用户确认测试
3. feedback capture -> reference -> rule review -> approved rule 生效闭环测试
4. PDF 中英双语输出与关键金额、币种、草案文案检查

### 5.2 结果

#### 5.2.1 真实业务单主链路

1. `orchestration_status: completed`
2. `draft_status: awaiting_user_decision`
3. `QuoteDocument` 生成成功
4. 审核级别输出为 `manager`
5. `Pending`、内部草案、review flags 等信息均按预期输出

#### 5.2.2 显式用户决策流

1. 未确认草案时，输出：
   - `draft_status: awaiting_user_decision`
   - `user_decision_prompt`
2. 用户传入 `user_decision: accept` 后，输出：
   - `draft_status: accepted`
   - `user_decision: accept`

#### 5.2.3 feedback 飞轮闭环

1. feedback 可写入 `.opencode/memory`
2. preference 可聚合并升级为 `active`
3. rule candidate 可升级为 `ready_for_review`
4. 审核后可写入 `.opencode/memory/approved-rules`
5. `quote_feedback_reference_skill` 可读取 `applicable_rules`
6. approved rule 可参与 `forbidden_patterns` / `review_alerts`

#### 5.2.4 PDF 输出专项

1. 中英双语 PDF 均可成功渲染
2. 顶部 `Currency` 显示正常
3. 明细金额和 summary 金额可显示币种符号
4. `Pending`、草案 remark、payment terms 均进入最终 HTML / PDF

### 5.3 结论

主报价流程、feedback 闭环、显式用户决策流、PDF 输出主能力均已达到可运行状态。

## 8. 当前已知风险与边界

虽然三轮测试整体通过，但仍需注意以下边界：

### 8.1 模板 smoke test 不等于模板业务正确性全覆盖

当前已证明各模板入口“能跑”，但还未对每个模板做完整业务正确性审查。

### 8.2 极简输入是“可运行”，不等于“输出总是业务充分”

当前系统对脏输入具有一定容错性，但极简输入下的报价合理性仍需结合业务判断。

### 8.3 `quote_feasibility_check_skill` 对 feedback memory 的消费仍较浅

目前 feedback 飞轮的主要收益仍集中在：

1. `quote_pricing_skill`
2. `quote_review_output_skill`

### 8.4 尚未做批量历史单回放

当前测试已覆盖代表性真实案例，但尚未做大规模多历史输入批量回放。

4. 增强 `feasibility` 阶段对 feedback memory 的消费

从三轮测试结果看，系统当前状态可以概括为：

1. **主链路稳定可运行**
2. **feedback 飞轮已形成可运行闭环**
3. **PDF 输出已达到可用状态**
4. **显式用户决策机制已落地**
5. **具备继续验收和试运行推进的基础**

因此，当前系统不应再被视为“原型设计稿”，而应视为“已具备主要功能可用性的智能报价系统初版”。
